计划员手册

English | 中文

您是生产计划员。您把需求转成已排程、已发布、可监控的工单。您依赖生产工程提供有效配方与路线， 并依赖质量工程确认检验准备。

最常用的屏幕

屏幕您在这里做什么
零件 与 BOM 主档确认拟计划的物料存在且结构正确。
配方在生成工单前确认所选配方/路线有效。
工单新建工单、检查生成的子单、设定数量与日期，并执行允许的状态动作。
计划按日期、机台组与计划视图排程工单。
队列系统确认已发布工单进入执行队列。
看板打开 OEE、KPI Production 和 Main Layout 入口；指标定义仍以业务负责人确认为准。
状态动作只在图标有标签或演示负责人确认后，才解释可见状态/动作图标。

新建工单前

  1. 确认零件与修订正确。
  2. 确认 BOM已准备好支持计划数量。
  3. 确认计划路线有对应配方。
  4. 确认所需机台组与班次可用。
  5. 如果工作需要检验，确认 QA 准备情况。

每日计划例行事项

时间动作
上班开始查看未结需求、今日到期工单与被阻塞工单。
发布前检查零件、BOM、配方、数量、交期与机台组。
班中关注队列系统，响应生产主管升级。
变更后重新检查计划，确保排程与现场一致。
下班前查看看板，并复查需要重排、取消或强制结束的工单。

升级规则

  • 配方、路线、机台能力、节拍或 NC 程序问题，升级给生产工程师。
  • 检验设置失败或缺失，升级给质量工程师。
  • 用户访问或权限缺失，升级给管理员。

常见问题

问题可能原因下一步
生成工单缺少预期子单配方或工序定义不完整请生产工程检查配方/工序设置
计划板为空日期范围、机台组或计划页面过滤排除了工单放宽过滤并检查工单状态
工单在计划中但不在队列中工单可能未发布，或队列页面过滤不同检查状态与队列过滤
工单卡在生产中子作业可能未关闭，或数量未完成先检查可见状态；在使用不清楚的动作图标前升级给主管
