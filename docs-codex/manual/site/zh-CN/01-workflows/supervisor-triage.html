生产主管延误作业排查

English | 中文

当班次中已释放作业缺失、延误、阻塞或不清楚时，请使用本页。

排查顺序

  1. 先看队列系统。
  2. 与工单对照。
  3. 在计划中确认排程。
  4. 在检验记录和 NCR 中检查质量状态。
  5. 只把生产看板作为高层级信号；OEE 定义仍需业务负责人确认。

看到这种情况时这样处理

情况先看页面检查内容操作 / 升级
队列中缺少作业Queue System筛选、机台/工作区域、零件、工序、状态与工单对照；如果 WO 未释放，升级给计划员。
WO 存在但未排程Planning日期范围、机台组、计划视图带 WO 和预期机台/工作区域升级给计划员。
队列行存在但无法开始Queue System可见状态、必要检验、阻塞步骤根据缺失条件升级给计划员、质量或生产工程师。
机台或程序不匹配Machines / NC Programs机台组、NC 程序准备情况升级给生产工程师。
检验失败或未保存Inspection RecordsWO/作业、结果、已保存记录升级给质量工程师。
NCR 未关闭NCR状态、受影响 WO/零件、必要跟进等质量确认下一步后再做生产决定。
OEE 或看板信号不清楚Dashboards页面、筛选、指标名称记录为待决事项；不要只凭不清楚指标采取行动。
用户看不到页面或操作Users and Roles用户、角色、可见菜单/操作升级给管理员。

延误决策表

收集可见证据后使用本表。本表用于安全分派：不要把未确认图标、不清楚看板指标或猜测的队列状态作为唯一行动依据。

症状可见证据可能负责人下一步安全动作不要做
队列行缺失队列筛选、WO/作业、零件/工序、机台/工作区域、日期，以及没有匹配行。如果 WO 未释放或未排程，负责人是计划员；如果访问隐藏 Queue System，负责人是管理员。对照 Production Orders 和 Planning，然后把 WO/作业、筛选和截图发给负责人。不要因为另一行排在第一位，就让操作员运行替代行。
WO 存在但未排程Production Orders 行存在；Planning 中没有匹配日期/区域的排程作业。计划员发送 WO、零件/工序、预期日期和预期机台/工作区域。不要从不清楚的状态动作释放、重置或关闭 WO。
队列行无法开始队列行存在，但状态映射到 Paused / blocked，行操作缺失，或必要检验/设置不清楚。根据缺失条件，可能是计划员、生产工程师、质量工程师或管理员。使用队列状态表，保留行/动作区域截图，并按缺失证据分派。不要绕过被阻塞的行，也不要推测隐藏的开始动作。
质量停止Inspection Records 或 SMARTQC 检验显示检验缺失、失败、不清楚或未保存。质量工程师暂停生产决定，直到质量确认检验结果、处置或返工要求。不要只凭已保存但不清楚的结果继续或完成生产。
NCR 未关闭NCR 行/状态影响同一 WO、零件、批次或工序。质量工程师记录 NCR 编号/状态，等待质量处置后再做生产动作。不要只凭看板或队列状态关闭、释放或继续。
看板不清楚OEE、KPI Production 或 Main Layout 指标定义不清、页面为空或筛选不匹配。演示负责人或业务负责人只把看板作为打开观察项；与 Production Orders、Planning 和 Queue System 对照。不要把 OEE/KPI 作为延误归属或生产动作的唯一原因。
访问缺失用户看不到预期侧边栏页面、工具栏动作、行操作或保存按钮。管理员保留用户、角色、预期页面/动作和当前截图；检查 Users and Roles。未经管理员或演示负责人批准，不要更改角色设置。

延误作业证据

当作业延误或看起来卡住时，请先收集以下可见字段，再判断责任归属。

证据先看哪里为什么重要
到期时间或生产日期工单、Planning确认该作业在当前复查窗口中是否真的延误。
计划开始/结束或计划区域Planning显示预期机台、组别、产线或日期。
队列状态Queue System把可见行或缺失行条件映射到 Ready、Active、Paused / blocked、Complete 或 Hidden / missing。
最后可见更新Queue System、工单帮助区分过期队列行和刚变更的作业。
产出与目标对比工单或指定报工页面显示延误是否因为数量仍未完成。
停机或停止原因指定报工页面或主管记录帮助把问题分派给生产、工程或维护。
负责人或当前角色工单、Queue System、Users and Roles显示谁可以执行下一步可见动作。

质量停止检查

停止条件检查内容下一步
缺少必要检验检验计划、SMARTQC 检验请质量工程师确认所需检验表或检验步骤。
已录入检验但结果不清楚检验记录不要根据不清楚结果释放或继续生产。
检验失败检验记录、NCR等质量确认处置后再做生产决定。
设备或校准影响结果设备校准接受测量前先升级给质量。

交接时记录什么

  • WO 或队列行。
  • 机台/工作区域。
  • 当前可见状态。
  • 发现问题的页面。
  • 当按钮、图标或状态不清楚时附截图。
  • 已升级给的人员或角色。
