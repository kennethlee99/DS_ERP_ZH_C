# 管理员设置检查清单

> [English](../../en/01-workflows/admin-setup-checklist.md) | 中文

在计划员、操作员或主管开始现场流程前，请使用本检查清单。本页区分手册可以确认的内容，以及仍需要负责人澄清的内容。

操作标签、用户-作业人员设置、权限标签、语言设置、看板定义和报工页面证据等未结事项，请查看[证据与待决事项登记](../00-open-decisions.md)。

> **待确认**
>
> | 当前可安全执行 | 尚未确认 | 确认负责人 | 需要的准确答案 |
> |---|---|---|---|
> | 复查可见用户、角色、侧边栏、菜单和截图，不重新设计访问权限。 | 用户-作业人员关系、权限标签、角色名称和语言设置。 | 管理员或实施负责人。 | 操作员使用登录用户、关联作业人员记录还是独立作业人员记录；哪些角色名称和权限标签对应每个必需页面/操作；语言设置由哪里控制。 |

## 安全快速开始

当您只需要让指定用户可以确认流程，而不是重新设计访问权限时，使用本表。

1. **开始条件：** 您有需要验证的培训用户或复查账号。
   **打开屏幕：** [用户和角色](../40-administration/users-and-roles.md)。
   **步骤：** 确认登录账号存在且可用。
   **预期结果：** 该账号可以识别，并可用于本次复查。
   **停止/联系条件：** 如果用户缺失、不可用或登录身份不明确，请停止并联系管理员。
2. **开始条件：** 账号存在。
   **打开屏幕：** [用户和角色](../40-administration/users-and-roles.md)。
   **步骤：** 确认用户拥有计划员、操作员、主管或管理员场景预期角色。
   **预期结果：** 用户的角色分配可见并已记录。
   **停止/联系条件：** 如果角色缺失、角色名称为 `待确认`，或权限标签未经负责人确认，请停止。
3. **开始条件：** 场景包含操作员执行或机台/工作区域分派。
   **打开屏幕：** [用户和角色](../40-administration/users-and-roles.md)，以及应用显示的任何作业人员或工作区域字段。
   **步骤：** 记录流程使用登录用户、关联作业人员，还是独立作业人员记录。
   **预期结果：** 分派身份已记录，可用于第一天场景。
   **停止/联系条件：** 如果用户-作业人员关系未确认，请停止并联系管理员或实施负责人。
4. **开始条件：** 角色和身份已记录。
   **打开屏幕：** 使用目标用户登录，然后打开该角色必需的侧边栏页面。
   **步骤：** 确认必需菜单和可见操作出现，不更改角色设置。
   **预期结果：** 角色必需页面和操作可见；缺失项目已作为证据记录。
   **停止/联系条件：** 如果菜单或操作隐藏、无标签或与预期场景不一致，请停止。
5. **开始条件：** 角色已经可以进入对应流程。
   **打开屏幕：** [零件](../20-engineering/parts.md)、[BOM](../20-engineering/bom.md)、[配方](../20-engineering/recipes.md)、[机台](../20-engineering/machines.md)、[NC 程序](../20-engineering/nc-programs.md)和相关流程页。
   **步骤：** 确认第一天场景需要的主数据和流程页面可见。
   **预期结果：** 计划员、操作员或主管可以使用已确认的可见页面进入角色流程。
   **停止/联系条件：** 如果指定物料的任何前提页面缺失或为空，请停止并联系“主数据准备”表中对应负责人。

## 用户和作业人员

当前手册无法证明 Users 和 Workers 是同一记录、关联记录还是分开设置。操作员流程开始前必须把这点作为实施决定。请记录用户登录账号、屏幕上显示的作业人员身份、机台/工作区域和角色。

## 角色和权限

使用[用户和角色](../40-administration/users-and-roles.md)确认可见访问。除非管理员或负责人确认含义，否则不要自行解释看起来像技术标签的权限文字。如果页面或操作缺失，请记录用户、角色、预期页面/操作和当前截图。

| 术语 | 实际含义 |
|---|---|
| 用户 | 用于登录网页应用的账号。 |
| 作业人员 | 如果现场流程使用，则表示车间身份或分派记录；在确认前，用户与作业人员的关系仍是待确认事项。 |
| 角色 | 分配给用户的可见访问分组。 |
| 权限标签 | Roles 页面上可见的标签，可能解释为什么某个菜单或操作出现。除非负责人确认，不要把它翻译成技术规则。 |
| 可见操作 | 登录用户能看到的按钮、图标、行操作、菜单项或弹窗操作。 |

## 翻译

手册本身是双语的，但可见语言和翻译管理页面仍是待决事项。在负责人确认预期语言流程前，不要声称已有完整翻译设置流程。

## 通用示例：管理员角色分配

本示例只说明应保留什么类型的证据。请用现场环境中真实可见的值替换占位内容。

| 字段 | 应保留的示例证据 |
|---|---|
| 用户身份 | 登录账号、如可见则包含显示名称、账号状态和预期场景角色。 |
| 角色分配 | 分配给用户的可见角色名称，以及显示该分配的角色页或用户详情截图。 |
| 访问检查 | 使用该用户登录后，场景需要的侧边栏页面和可见操作。 |
| 权限证据 | 按屏幕原文记录权限标签；在管理员确认映射前，不把它解释成规则。 |
| 停止/联系条件 | 如果角色名称、权限标签、用户-作业人员关系或语言设置不明确，保持为 `待确认` 并联系管理员或实施负责人。 |

## 主数据准备

| 如果缺少 | 受影响流程 | 联系对象 |
|---|---|---|
| [零件](../20-engineering/parts.md) | 计划员无法可靠建立或确认 WO。 | 生产工程师或计划负责人 |
| [BOM](../20-engineering/bom.md) | 计划员无法确认生产结构。 | 生产工程师 |
| [配方](../20-engineering/recipes.md) | 计划员可能生成或释放错误作业。 | 生产工程师 |
| [机台](../20-engineering/machines.md) | 计划和队列分配可能错误。 | 生产工程师 |
| [NC 程序](../20-engineering/nc-programs.md) | 操作员可能没有预期执行程序。 | 生产工程师 |
| [检验计划](../30-quality/inspection-planning.md)或[检验表](../35-smartqc/check-sheets.md) | 作业可能尚未具备质量准备。 | 质量工程师 |
