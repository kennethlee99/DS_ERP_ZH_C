# 操作员运行下一个作业流程

> [English](../../en/01-workflows/operator-run-next-job.md) | 中文

当操作员需要知道下一步运行哪个作业，以及如何确认作业已达到可交接状态时，请使用本流程。

派工规则、行开始操作、生产报工、检验确认和交接信号的未结事项记录在[证据与待决事项登记](../00-open-decisions.md)中。

> **待确认**
>
> | 当前可安全执行 | 尚未确认 | 确认负责人 | 需要的准确答案 |
> |---|---|---|---|
> | 筛选队列、把队列行与指定 WO/作业匹配、记录可见证据，并在任何不明确的开始/报工/完成操作前停止。 | 队列派工规则、队列开始操作、生产报工页面和完成/交接信号。 | 生产主管；报工路径需要时由计划负责人共同确认。 | 多行可见时用哪条规则选择正确行、哪个可见行操作打开或开始作业、哪个报工页面记录产出/停机，以及什么可见信号证明完成或交接。 |

## 目标

找到下一个作业，确认它已经准备好，按现场已确认的流程开始作业，通过指定报工页面记录进度，完成必要检验，并确认结果已保存。

## 操作流程

| 步骤  | 打开                                                   | 操作                              | 进入下一步前检查                    |
| --- | ---------------------------------------------------- | ------------------------------- | --------------------------- |
| 1   | [队列系统](../10-production/queue-system.md)             | 选择现场流程使用的机台、产线、工作区域、日期或状态筛选。      | 该行属于您的机台/工作区域，并与分配的零件/工序一致。 |
| 2   | [队列系统](../10-production/queue-system.md)             | 只有在负责人或主管确认派工规则后，才选择第一行。      | 除非规则已说明，否则不要从行顺序假设优先级。      |
| 3   | [工单](../10-production/production-orders.md)          | 如果队列行不明确，查找 WO 并比较零件、数量、到期日和工序。 | WO 状态支持车间执行。                |
| 4   | [队列系统](../10-production/queue-system.md)             | 使用可见行操作打开作业或下一步工作步骤。            | 如果开始操作没有标签，请请求带操作名称的截图。     |
| 5   | 指定生产报工页面                                             | 只在本部署指定的页面记录产出、停机或问题说明。         | 离开页面前可以看到保存或确认信号。           |
| 6   | [SMARTQC 检验](../35-smartqc/inspection-data-entry.md) | 如果需要检验，使用作业行中显示的作业、阶段、批次或日期搜索。  | 必填测量值已保存，结果可见。              |
| 7   | [检验记录](../30-quality/inspection-records.md)          | 当质量状态影响释放或交接时，确认已保存的检验记录。       | 记录属于同一个 WO/作业，并显示预期结果。      |

## 操作员标准操作路径

在把作业视为已开始、已报工、已检验或已完成前，先使用本表。未知的本地控制仍保留为明确阻塞项，直到负责人提供带标签截图。

| 步骤 | 精确页面 / 路径 | 选择内容 | 精确操作 / 控制 | 保存信号 | 完成 / 交接信号 |
|---|---|---|---|---|---|
| 筛选队列 | [Queue System](../10-production/queue-system.md)（`Planner / Queue System`，`<APP_BASE_URL>/Production/QueuePage`） | 现场流程或主管指定的机台、产线、工作区域、日期和状态筛选。 | 只使用可见筛选；不要只按行顺序选择。 | 刷新后筛选后的行列表仍可见。 | 匹配指定 WO/作业、零件、工序和工作区域的行映射到[队列状态表](../10-production/queue-system.md#队列状态表)中的 `Ready` 工作术语。 |
| 应用派工规则 | [Queue System](../10-production/queue-system.md) | 主管/现场流程指定的行，或匹配已确认派工规则的行。 | 如果规则依赖行顺序、优先级或未标注状态，则需截图确认。 | 主管或现场流程确认所选 WO/作业。 | 所选行已记录 WO/作业、零件/工序、工作区域和可见状态。 |
| 打开或开始作业 | [Queue System](../10-production/queue-system.md) | 已确认的队列行。 | 需截图确认：精确行打开/开始操作标签仍需截图。 | 操作后可见开始确认或状态变化。 | 队列行或作业页面显示 `Active` 工作状态，或主管确认当前可见状态。 |
| 报告进度/产出/停机 | 指定生产报工页面 | 同一 WO/作业、工序、机台/工作区域、产出数量、停机原因和需要时的问题说明。 | 需截图确认：报工页面路径/URL、字段和保存操作仍需截图。 | 保存/确认消息、更新后的行或时间戳可见。 | 产出、停机或备注证据已保留给主管交接。 |
| 完成检验 | [SMARTQC Inspection](../35-smartqc/inspection-data-entry.md) | 队列/作业行中的作业、阶段、批次或日期。 | 输入必填测量值并使用可见保存操作。 | 保存后结果或更新时间可见。 | [Inspection Records](../30-quality/inspection-records.md) 显示同一 WO/作业/阶段和预期结果。 |
| 完成或交接 | [Queue System](../10-production/queue-system.md)、[Production Orders](../10-production/production-orders.md) 或指定报工页面 | 刷新后的同一 WO/作业和工序。 | 只使用已确认的完成/交接操作；否则联系主管处理。 | 状态变化、已保存记录或主管确认可见。 | 队列/WO 状态映射到 `Complete`、`Active` 或 `Paused / blocked`；交接前记录未完成工作。 |

## 如何选择下一个作业

除非主管或负责人明确说明行顺序就是派工规则，否则不要只按行顺序选择作业。

| 选择规则 | 何时使用 | 检查内容 |
|---|---|---|
| 指定机台或工作区域 | 操作员负责一台机台、一条产线或一个工位。 | 队列行的机台/工作区域与分配一致。 |
| 计划日期或班次 | 班次计划就是派工规则。 | 行日期或计划期间与当前班次一致。 |
| 主管指令 | 多个准备作业同时可见。 | 主管指定要运行的 WO、零件、工序或机台。 |
| 质量保留已解除 | 作业曾因检验或 NCR 被阻塞。 | [检验记录](../30-quality/inspection-records.md)或 [NCR](../30-quality/ncr-non-conformance.md) 显示该问题不再阻塞作业。 |
| 现场流程顺序 | 内部流程有固定场景。 | 使用现场流程中的 WO 或零件号，不要只用行位置。 |

## 操作记录

把作业视为已开始、已报工或已完成前，请填写本表。

| 阶段 | 使用的可见操作 | 保存/确认信号 | 保留证据 |
|---|---|---|---|
| 开始 |  |  | 带操作标签的队列行截图 |
| 进度 / 产出 |  |  | 报工页面、数量、停机或备注截图 |
| 检验 |  |  | [SMARTQC 检验](../35-smartqc/inspection-data-entry.md)结果或[检验记录](../30-quality/inspection-records.md)截图 |
| 完成 / 交接 |  |  | 刷新后的队列或 WO 状态 |

## 通用示例

以下示例只说明应保留什么类型的证据。请用现场流程中真实可见的值替换占位内容；不要编造屏幕上没有的标签。

### 准备好的队列行

| 字段 | 应保留的示例证据 |
|---|---|
| 队列筛选 | [队列系统](../10-production/queue-system.md)上可见的机台、产线、工作区域、日期和状态筛选。 |
| 行身份 | 与主管分派一致的 WO/作业、零件、工序和机台/工作区域。 |
| 安全操作 | 只有派工规则已确认时，选行才安全；只有可见队列开始操作已确认时，才能打开或开始作业。 |
| 停止/联系条件 | 如果多行都符合但没有派工规则、开始操作没有标签，或只有行顺序能说明优先级，请联系主管处理。 |

### 生产报工和交接证据

| 字段 | 应保留的示例证据 |
|---|---|
| 报工上下文 | 指定报工页面上可见的同一 WO/作业、工序、机台/工作区域和登录角色。 |
| 已记录事件 | 只在部署指定页面记录的产出数量、停机原因、问题说明或未完成工作说明。 |
| 保存证据 | 操作后的确认消息、时间戳、更新后的行或已保存记录。 |
| 交接证据 | 刷新后的队列/WO 状态，以及下一班需要知道的未完成数量、停机或检验跟进。 |

## 完成信号

| 信号             | 含义                                |
| -------------- | --------------------------------- |
| 队列行不再出现在准备列表中  | 作业可能已移动到其他状态或筛选结果。确认前不要直接假设完成。    |
| WO 或队列状态显示完成状态 | 当前步骤可能已经完成。请与 WO 和工序进行匹配。         |
| 检验结果已保存        | 质量录入已记录，但生产完成仍取决于 WO 或队列状态。       |
| Save 不可用       | 可能有必填字段为空，或角色不允许操作。先检查可见必填字段，再联系主管或管理员处理。 |

## 停止并上报

| 如果看到... | 联系对象 | 提供信息 |
|---|---|---|
| 多个准备作业但没有派工规则 | 主管 | 工作区域、可见行和当前筛选 |
| 开始作业的操作不明确 | 主管 | 队列行和工具栏截图 |
| 未明确进度/产出页面 | 主管或计划员 | WO、机台/工作区域和当前页面 |
| 需要检验但找不到记录 | 质量工程师 | WO、作业/阶段、批次/日期和截图 |

## 截图请求

本流程仍需要分别收集队列派工规则和队列开始操作的证据。派工证据应显示筛选、候选行、WO/作业身份，以及负责人确认的选行规则。开始操作证据应显示登录角色、[Queue System](../10-production/queue-system.md) 筛选、已选 WO/作业行、可见打开/开始标签，以及操作后的保存、页面或状态信号。进度/产出报工和完成确认也仍未结。缺少的证据记录在[证据与待决事项登记](../00-open-decisions.md)中。
