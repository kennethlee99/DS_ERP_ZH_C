Path: Planner / Production Orders

## 页面用途

Production Orders 用于查找和确认生产工单。只有在操作标签和执行结果已确认时，才使用状态操作。计划员和生产主管通常先从这里确认作业是否已准备好进入车间执行。

释放、暂停、取消、重置、关闭和其他状态操作在可见标签和结果确认前，仍记录在[证据与待决事项登记](../00-open-decisions.md)中。

## 页面显示内容

- 可搜索的工单列表，包含状态、数量、物料、日期和负责人等信息。
- 当现场流程要求新建工单时，用于创建工单的可见 `New Production Orders` 按钮。
- 在账号允许时显示的编辑、导出、刷新和状态变更按钮。
- 可按工单、日期、零件、状态或相关生产信息缩小范围的筛选条件。
- 可打开所选工单进行复查的行操作。

## 常用操作

1. 按正在复查的工单或日期范围筛选。
2. 打开行并确认物料、数量、到期日和状态。
3. 只有在计划、队列和质量准备情况确认后，才使用屏幕上允许的状态操作。如果操作是不带标签的图标，请先把它作为截图验证需求，不要直接写成明确点击步骤。
4. 变更后回到列表，确认该行显示预期状态。

## 要检查什么

- 工单属于正确的零件和版本。
- 计划数量和日期符合现场场景。
- 状态支持下一步，用户才应在 Planning 或 Queue System 中查找它。
- 子作业、检验或工程准备没有阻塞释放。

## 常见问题

| 问题 | 含义 |
|---|---|
| 找不到工单 | 筛选条件可能太窄，或该场景尚未建立工单。 |
| 状态操作不可用 | 工单可能不在允许执行该操作的状态。 |
| 已释放工单不在队列中 | 重新检查工单状态、队列筛选和计划工作区域。 |
| 操作图标含义不明确 | 在把它写成释放或状态变更步骤前，请求带标签的工具栏截图。 |
